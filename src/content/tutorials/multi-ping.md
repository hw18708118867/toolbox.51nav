## Ping 到底在测什么？

Ping 是网络诊断最古老的工具之一，1983 年由 Mike Muuss 写出来，名字灵感来自声呐的"ping"声。它测的是**两台主机之间的可达性和往返延迟（RTT, Round-Trip Time）**。

原理基于 **ICMP 协议**（Internet Control Message Protocol，互联网控制消息协议）。Ping 发一个 ICMP Echo Request（Type 8）给目标，目标收到后回一个 ICMP Echo Reply（Type 0），中间掐表计时，差值就是 RTT。

注意 ICMP 工作在网络层（IP 层），**不走 TCP/UDP，不建立连接，不传数据**。所以 Ping 通只能证明"网络层可达"，不能证明"HTTP 服务正常"——服务器进程挂了但网络层还在，Ping 照样通。这是 Ping 的根本局限。

## RTT 延迟由什么构成

一个往返延迟拆开看：

```
RTT = 传播延迟 + 传输延迟 + 处理延迟 + 排队延迟
```

- **传播延迟**：信号在光缆/铜线里跑的物理时间，受光速限制。这是延迟的大头，和距离成正比——北京到纽约单程光纤传播约 70ms，往返就 140ms，这是物理极限，再优化也绕不开
- **传输延迟**：把数据包比特推上链路的时间，和包大小/带宽有关，通常很小
- **处理延迟**：路由器转发时查路由表、TTL 减一等的时间
- **排队延迟**：路由器端口拥塞时包排队等候的时间，波动大，是延迟抖动的主要来源

这就解释了为什么**物理距离是延迟的硬下限**：跨洋访问再快也快不过光速。要降延迟，最有效的办法是**把服务器搬近用户**——这正是 CDN 存在的根本理由。

## 为什么需要"多地" Ping

单点 Ping 只告诉你"从我这台机器到目标多快"。但你的用户分布在全球，从北京 Ping 快不代表从纽约、从孟买也快。多地 Ping 从全球多个节点同时测，能看到：

1. **全球可达性**：有没有某些地区完全不通（被墙、路由黑洞、机房故障）
2. **地区延迟分布**：哪些地区访问慢，需要加 CDN 节点
3. **CDN 效果验证**：开了 CDN 后各地延迟应该普遍下降，对比前后数据
4. **机房选址参考**：选机房时看目标用户群的延迟表现
5. **故障定位**：某地区突然延迟飙升或丢包，能快速定位是哪个环节的问题

典型场景：你部署在上海机房，北京用户 Ping 30ms 很正常，但纽约用户 Ping 250ms 也正常（跨太平洋）。如果纽约用户反馈"卡"，多地 Ping 一测发现是 280ms，那就不是 bug 是物理距离——解决方案是上 CDN 或加海外节点，而不是排查服务器。

## 怎么解读 Ping 结果

一次 Ping 通常会输出几组数据，关键指标：

| 指标 | 含义 | 关注点 |
|------|------|--------|
| 最小 RTT | 最好的一次 | 反映理论上限 |
| 平均 RTT | 所有轮次平均 | 综合参考 |
| 最大 RTT | 最差的一次 | 偶发抖动 |
| 抖动（jitter） | RTT 的波动幅度 | 越小越稳定，对实时应用（语音/游戏）关键 |
| 丢包率 | 没收到 Reply 的比例 | >0 就要警惕，>5% 明显影响体验 |

解读要点：

- **丢包比延迟更致命**：延迟 200ms 但 0 丢包，网页能开；延迟 30ms 但 20% 丢包，网页卡得没法用。TCP 把丢包当拥塞信号，会疯狂降速
- **抖动大说明链路不稳**：min/avg/max 接近说明稳定，差距大说明有拥塞或路由抖动
- **偶尔超时不一定是故障**：单次超时可能是瞬时拥塞，连续超时才是真问题

## Ping 的局限和"假 Ping"

1. **ICMP 被限速/屏蔽**：很多服务器和防火墙为防 DDoS 会限制或丢弃 ICMP，导致 Ping 不通但 HTTP 正常。所以"Ping 不通 ≠ 服务挂了"，反过来要测服务可用性得用 HTTP 探测（curl/HTTP 状态码）
2. **CDN 的特殊性**：Ping 一个 CDN 域名，DNS 会把请求解析到离 Ping 节点最近的 CDN 边缘节点。所以多地 Ping CDN 域名，各节点延迟都低（各自就近），这正好验证了 CDN 生效
3. **测的是网络不是应用**：服务器 CPU 跑满、数据库锁死，Ping 照样通。要测应用层健康，得用 HTTP 健康检查

## 延迟的常见量级参考

| 场景 | 典型 RTT |
|------|---------|
| 同城/同机房 | < 2ms |
| 同省/同城骨干 | 5~15ms |
| 跨省国内 | 20~50ms |
| 跨洲（亚欧/亚美） | 150~250ms |
| 卫星链路 | 500ms+ |

国内访问国内站点超过 50ms 通常就有优化空间（可能路由绕远了）。全球分布的服务，目标是把各地延迟压到 100ms 以内，超出就得靠 CDN。

## 小结

Ping 是基于 ICMP 的网络层可达性和延迟测量，RTT 的下限由光速决定、所以物理距离是硬约束。多地 Ping 的价值在于提供"全球视角"，让你看到不同地区用户的真实体验，验证 CDN 效果、定位区域故障。记住两个关键认知：Ping 通不代表服务正常（只测网络层），丢包比延迟更影响体验。要排查具体某地的解析和归属，配合 [[dns-lookup]] 和 [[ip-lookup]]。
